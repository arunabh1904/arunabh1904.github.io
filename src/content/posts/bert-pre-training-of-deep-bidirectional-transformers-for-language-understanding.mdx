### tiny_pretrain.py – core BERT objectives in ~40 LOC
```python
import random, torch
from typing import Iterable, Tuple, Generator
from transformers import BertTokenizer

def mask_tokens(ids: torch.Tensor | list[int], tok: BertTokenizer,
                p: float = 0.15) -> Tuple[torch.Tensor, torch.Tensor]:
    ids = torch.tensor(ids)
    is_special = torch.tensor(tok.get_special_tokens_mask(ids.tolist(), True)).bool()
    mask = ~is_special & (torch.rand_like(ids.float()) < p)

    labels = torch.where(mask, ids, torch.full_like(ids, -100))
    rand = torch.randint(len(tok), ids.shape, dtype=torch.long)

    ids[mask] = torch.where(torch.rand_like(ids.float()) < 0.8,
                            tok.mask_token_id,
                            ids[mask])
    ids[(ids == tok.mask_token_id) & (torch.rand_like(ids.float()) < 0.125)] = \
        rand[(ids == tok.mask_token_id) & (torch.rand_like(ids.float()) < 0.125)]
    return ids, labels


def make_sentence_pairs(docs: Iterable[str], ratio: float = 0.5
                        ) -> Generator[Tuple[str, str, int], None, None]:
    docs = [d.split('.') for d in docs]
    for sents in docs:
        sents = [s.strip() for s in sents if s.strip()]
        for i in range(len(sents)-1):
            a, b_true = sents[i], sents[i+1]
            if random.random() < ratio:
                b = random.choice(random.choice(docs)).strip()
                yield a, b, 1
            else:
                yield a, b_true, 0
```

### BERT, Six Years On: What Still Matters

BERT's recipe is simple enough to summarize and strong enough to keep reappearing: pre-train a Transformer encoder on large text corpora, then fine-tune the same backbone for many downstream tasks. The base model has 12 layers, the large model has 24, and both were trained on roughly 3.3B tokens.

---

## How the Pre-training Works

### Masked-Language Modelling (MLM)

The `mask_tokens` function corrupts 15% of input pieces at random, forcing the network to infer each blank from both left and right context. This made BERT genuinely bidirectional, unlike GPT-1's left-to-right objective or ELMo's concatenation of two one-directional RNNs.

### Next-Sentence Prediction (NSP)

`make_sentence_pairs` creates 50% real A to B pairs and 50% mismatches. The classifier sitting on `[CLS]` learns a discourse-level coherence signal. Later work, especially RoBERTa, showed that NSP was not necessary for strong accuracy, but the idea still seeded a large family of inter-sentence objectives for sentence embeddings and dense retrieval.

## Why BERT Hit Hard

BERT combined four things at the right time. Masked-language modeling gave the encoder access to both left and right context. A single backbone could support QA, NLI, NER, and sentiment by swapping a task head and fine-tuning end to end. `[CLS]` and segment embeddings gave pairwise classification and retrieval a default recipe. Open weights and code turned the paper from a result into an ecosystem, leading quickly to RoBERTa, ALBERT, DistilBERT, and many domain-specific variants.

---

## Where It Falls Short

BERT was compute-hungry in 2018 and still non-trivial to reproduce casually. NSP did not age well; later ablations showed that many variants can drop it. The 512-token limit and quadratic attention make long documents awkward without architectural changes. BERT also has static knowledge, so domain adaptation or continual learning is required when the world or target domain moves.

---

## Looking Forward

BERT's recipe became a floor, not a ceiling. Better corruption schemes, cheaper compressors, and longer-context attention methods all stand on top of the same basic idea: mask some tokens, optionally test sentence order, backpropagate, and reuse the encoder everywhere. That elegance is why BERT is still worth teaching.
